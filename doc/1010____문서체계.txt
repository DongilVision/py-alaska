1. 문서 번호 규칙
----------------------------------------------------------------------------

1.1 번호 구조

    각 문서는 4자리 번호를 부여받으며, 천 단위로 분류 영역을 나눈다.
    같은 분류 내 문서는 10번 단위로 배치하여 향후 삽입 여유를 확보한다.

        ┌─────────────────────────────────────────────────┐
        │  번호 형식:  XYYY                               │
        │                                                  │
        │  X    = 분류 영역 (1~9)                          │
        │  YYY  = 영역 내 순번 (10 단위 간격)              │
        │                                                  │
        │  파일명:  XYYY____제목.txt                       │
        │  예시:    3060____IPC설계.txt                    │
        └─────────────────────────────────────────────────┘

1.2 파일명 규칙

    번호____제목.txt

    - 번호와 제목 사이: 언더스코어 4개 (____) 구분
    - 제목: 한국어 또는 영문, 공백 없이 작성
    - 확장자: .txt


2. 분류 영역
----------------------------------------------------------------------------

    번호대     분류              설명
    ─────────  ────────────────  ──────────────────────────────────────
    1xxx       가이드 / 총괄     시스템 구조 설명, 개발자 가이드, 문서 체계
    2xxx       Task              Task 구조 설계 (생명주기, 동작, 관리)
    3xxx       통신 (IPC/Signal) 통신 구조 설계 (IPC, Signal, Dispatcher)
    4xxx       구현 인프라        실행 기반 설계 (공유메모리, 프로퍼티)
    5xxx       디버그 기능        디버깅·모니터링 도구 설계
    9xxx       관리 / 기타        수정 이력, 향후 구현 계획

    ※ 6xxx, 7xxx, 8xxx 는 미사용 (향후 확장 예약)

4. 문서 간 참조 규칙
----------------------------------------------------------------------------

4.1 참조 표기

    문서 내에서 다른 문서를 참조할 때는 번호.제목 형식을 사용한다:

        관련: doc/3060____IPC설계.txt (상세 IPC 설계)
              doc/3070____Signal설계.txt (상세 Signal 설계)

    또는 본문 내에서:

        → 상세 내용은 3060.IPC설계 참조

4.2 문서 이동 시

    문서 번호를 변경할 때는 다음을 반드시 확인한다:

    1. 해당 문서를 참조하는 모든 문서의 참조를 갱신
    2. 해당 문서 내부의 자기 참조(footer 등)를 갱신
    3. grep으로 이전 번호 잔존 여부를 확인


5. 문서 작성 형식
----------------------------------------------------------------------------

5.1 공통 구조

    ============================================================
      ALASKA v2.6 - [문서 제목]
      Date: YYYY-MM-DD
    ============================================================

    1. 섹션 제목
    ------------------------------------------------------------
    1.1 소제목
        본문 ...

    N. 상태
    ------------------------------------------------------------
        상태: 확정 (v2.6)
        등록일: YYYY-MM-DD
        분류: [분류명]
        관련: doc/XXXX____제목.txt (설명)

5.2 작성 원칙

    - 한국어로 작성 (제목, 내용 모두)
    - ASCII 박스/테이블은 전각 또는 유니코드 라인문자 사용
    - 코드 예시는 들여쓰기 8칸으로 표시
    - 1000~1500줄 이내로 유지
