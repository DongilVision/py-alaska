# [ALASKA v2.3] 초경량·저지연 QoS 통합 상세 설계서 (Final Implementation Spec)

본 문서는 시그널 QoS, 자가 치유, 실시간 시각화 기능을 포함한 ALASKA v2.3 프레임워크의 상세 클래스 설계 및 구현 로드맵을 기술한다.

---

## 1. 클래스 구조 및 핵심 메서드 (Class Specification)

### 1.1 `QoSManager` (New Module)
QoS 프로필과 정책을 총괄 관리하는 싱글톤 엔진.
*   `load_levels(config)`: `config.json`의 QoS 레벨 로딩 및 SM 게시.
*   `get_profile(level_name_or_dict)`: 입력된 QoS 정보를 표준 ID로 변환.
*   `evaluate_violation(sig_id, elapsed)`: 데드라인 위반 시 정책에 따른 액션(HALT, RESTART 등) 실행.

### 1.2 `SmSignalRegistry` (Shared Memory)
*   `update_heartbeat(task_id)`: 원자적으로 `last_alive` 타임스탬프 갱신.
*   `get_subscribers_bitmap(sig_hash)`: 비트 연산을 위한 구독 태스크 비트맵 반환.
*   `get_architecture_map()`: 초기화 시 생성된 Mermaid 포맷 문자열 반환.

### 1.3 `VisualizerEngine` (Debugger Core)
*   `generate_static_map(tasks)`: 태스크 간 의존성을 분석하여 그래프 생성.
*   `push_sequence_event(event)`: 런타임 이벤트를 시퀀스 차트용 시계열 데이터로 변환.

## 2. 핵심 알고리즘 상세 (Detailed Algorithms)

### 2.1 Adaptive Quota 스케줄링 (Anti-Lag)
```python
def adaptive_recv_loop(self):
    weights = self.get_weights() # [20, 10, 5, 2, 1]
    while self.running:
        burst_start = time.perf_counter()
        for p, quota in enumerate(weights):
            for _ in range(quota):
                # 1. 처리 시간 예산 체크 (랙 방지 핵심)
                if (time.perf_counter() - burst_start) > 0.005: # 5ms limit
                    yield_to_os()
                    burst_start = time.perf_counter() # 예산 초기화
                
                # 2. 데이터 인출 및 처리
                data = self.queues[p].get_nowait()
                if data: self.dispatch(data)
                else: break
```

### 2.2 하트비트 기반 자가 치유 (Watchdog)
```python
def watchdog_monitor(self):
    for task in self.registry.tasks:
        if (time.time() - task.last_alive) > 2.0:
            if task.is_critical:
                self.signal.emit("system.halt", reason=f"Critical task {task.id} frozen")
            else:
                self.manager.restart_task(task.id)
```

## 3. 구현 계획 (Implementation Roadmap)

### Phase 1: 인프라 및 QoS 기초 (4일)
1.  **SM 통합:** `Manager.dict`를 `SmSignalRegistry`로 전면 교체.
2.  **QoS 프로필 엔진:** 딕셔너리 기반 QoS 해석 및 ID 매핑 로직 구현.
3.  **P2P 멀티큐:** 5단계 우선순위 큐 및 기초 스케줄러 구축.

### Phase 2: 자가 치유 및 정책 엔진 (3일)
1.  **전역 와치독:** 공유 메모리 하트비트 및 `psutil` 연동 모니터 구현.
2.  **세이프티 정책:** 데드라인 위반 시 액션(HALT, RESTART) 자동 실행 엔진 완성.
3.  **리소스 GC:** 죽은 태스크 자원 강제 회수 로직 통합.

### Phase 3: 가시성 고도화 (3일)
1.  **정적 맵 생성:** `TaskManager` 초기화 시 Mermaid 아키텍처 다이어그램 자동 생성.
2.  **시퀀스 차트:** 체인 ID 기반의 실시간 UML 트레이싱 UI 구현.
3.  **CPA 진단:** 웹 디버거 내 임계 경로 분석 및 지연 구간 강조 기능 추가.

### Phase 4: 안정화 및 벤치마킹 (2일)
1.  **병렬 스트레스 테스트:** 64개 이상의 태스크를 동시 가동하여 랙 및 크래시 여부 검증.
2.  **성능 리포트:** v2.1 대비 지연 시간 및 메모리 효율성 비교 데이터 산출.

---

## 4. 최종 결론
v2.3은 ALASKA 프레임워크의 기술적 정점이다. 고도로 최적화된 공유 메모리 인프라 위에 와치독과 QoS 정책을 결합함으로써, 어떠한 가혹한 환경에서도 **"시스템은 멈추지 않고, 데이터는 늦지 않는다"**는 약속을 지킬 수 있는 완벽한 제어 솔루션을 제공할 것이다.
