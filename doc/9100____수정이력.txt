■ InvokeDispenser CPU 절감 최적화 (4대 최적화)

1. RMI 응답 대기 — Spin-wait → Kernel Event (task_manager.py)
   - 변경 전: RmiClient._call()이 response_q.read()를 폴링 (CPU 1코어 점유)
   - 변경 후: response_event.wait(timeout)로 커널 대기 (CPU 0%)
   - 추가 자원: 태스크당 response_event (SmKernelEvent)
   - InvokeDispenser: 응답 기록 후 caller_ti.response_event.set()
   - 수정 파일: task_manager.py (TaskInfo, RmiClient, InvokeDispenser, TaskRuntime)

2. Signal Direct Emit — SEND 위임 제거 (task_signal.py)
   - 변경 전: emit() → send_queue → InvokeDispenser ① SEND → broker.emit()
   - 변경 후: emit() → broker.emit() (caller 스레드에서 직접)
   - SEND 단계 삭제. send_queue 제거. 디스패처 루프 4단계 → 3단계
   - 수정 파일: task_signal.py (SignalClient), task_manager.py (InvokeDispenser)

3. 조건부 notify — is_sleeping 플래그 (task_manager.py, task_signal.py)
   - 변경 전: 모든 시그널/RMI 전송마다 notify_event.set() (커널 syscall)
   - 변경 후: is_sleeping == 1일 때만 set() 호출 (메모리 읽기 ~50ns)
   - 추가 자원: 태스크당 is_sleeping (SmValue 'B', 1바이트)
   - 수정 파일: task_manager.py, task_signal.py (SignalBroker._notify_task)

4. 시간 호출 간소화 (task_manager.py)
   - 변경 전: RECV burst guard에 perf_counter() 호출 (최대 38회/cycle)
   - 변경 후: 카운트 기반 burst guard (50건/cycle, 시간 호출 0회)

   - @task       : on_* 메서드 자동 시그널 구독 데코레이터
   - @ui_thread  : Qt 메인 스레드 전환 데코레이터
