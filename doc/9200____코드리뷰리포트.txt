■ B01 [Critical] SmQueue FileExistsError → AttributeError 크래시

  위치: sm_queue.py:86
  증상: v2.6에서 numpy 제거 후 _write_idx 속성 없음 → 재시작 시 크래시
  수정: _write_idx[0] = 0; _read_idx[0] = 0 → self._rb.reset()

■ B03 [High] SmRingBuffer close() 후 Access Violation

  위치: sm_ring_buffer.py:214
  증상: _buf(memoryview)가 해제된 SHM 메모리를 참조 → 간헐적 크래시
  수정: _buf.release() + _buf = None 가드 추가

■ B04 [High] remove_task() 커널 핸들/SHM 누수

  위치: task_manager.py:1483
  증상: response_event, is_sleeping close 누락 → 장시간 운용 시 핸들 고갈
  수정: ti.response_event.close(), ti.is_sleeping.close() 추가

■ B05 [High] emit() 부분 전달 중단

  위치: task_signal.py:581
  증상: 첫 subscriber 큐 full 시 return False → 나머지 미전달
  수정: _all_ok = False; continue → 전원 전달 후 최종 결과 반환

■ B06 [High] unregister() 좀비 구독

  위치: task_signal.py:689
  증상: Manager.dict만 제거, SmSignalRegistry 비트맵 미해제 → 닫힌 큐에 write
  수정: sm_reg.unsubscribe() + 캐시 무효화 추가
