=== RmiPort 기반 RMI 아키텍처 ===

1. RmiPort 구조
   - task_name: TASK 이름
   - tx_queue: 응답 수신용 큐 (내가 보낸 요청에 대한 응답)
   - rx_queue: 요청 수신용 큐 (다른 Task로부터의 요청)

2. RmiPortMap
   - Dict[str, RmiPort] 형태
   - 모든 TASK의 RmiPort를 담고 있음
   - 예: {"task_a": RmiPort(...), "task_b": RmiPort(...)}

3. TaskManager 초기화 흐름
   1) task_map에서 모든 TASK 이름 수집
   2) 각 TASK별 RmiPort 생성 (tx_queue, rx_queue)
   3) RmiPortMap 구성
   4) Task 인스턴스 생성 시 RmiPortMap 전달

4. RMI 호출 흐름 (TaskA -> TaskB)
   1) TaskA가 TaskB의 rx_queue에 요청 전송
   2) 요청에 TaskA의 tx_queue 포함 (응답 수신용)
   3) TaskB가 요청 처리 후 TaskA의 tx_queue로 응답 전송
   4) TaskA가 자신의 tx_queue에서 응답 수신

5. 장점
   - RMIBroker 불필요 (중앙 집중 제거)
   - 직접 큐 접근으로 성능 향상
   - 요청별 응답 큐 생성 불필요 (tx_queue 재사용)
   - 구조 단순화

6. 코드 예시
   @dataclass
   class RmiPort:
       task_name: str
       tx_queue: Queue  # 응답 수신
       rx_queue: Queue  # 요청 수신

   RmiPortMap = Dict[str, RmiPort]

   class Task:
       def __init__(self, name: str, rmi_port_map: RmiPortMap, ...):
           self._rmi_port_map = rmi_port_map
           self._my_port = rmi_port_map[name]

       def call_rmi(self, target: str, method: str, *args):
           target_port = self._rmi_port_map[target]
           request = RMIRequest(
               response_queue=self._my_port.tx_queue,  # 내 tx_queue로 응답
               ...
           )
           target_port.rx_queue.put(request)  # 상대 rx_queue로 요청
