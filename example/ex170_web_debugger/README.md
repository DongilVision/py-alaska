# ex170_web_debugger - 실시간 모니터링 및 정책 검증

## 개요
ALASKA v2.1의 **분산 P2P 통신 성능**과 **2KB 하드 리미트 정책**을 웹 브라우저를 통해 실시간으로 확인하는 예제입니다.

## 주요 학습 포인트
1. **Web Inspector 접속:** `http://localhost:7000`에 접속하여 전체 태스크의 헬스 체크 및 시그널 통계를 볼 수 있습니다.
2. **P2P 시그널 통계:** 어느 시그널이 초당 몇 번(Hz) 발생하는지, 마지막 발생 시점은 언제인지 웹 UI에서 실시간 업데이트됩니다.
3. **2KB 제한 정책:** `SensorTask`가 2KB가 넘는 데이터를 시그널로 보내려 할 때 프레임워크가 이를 즉각 차단하고 `PayloadTooLargeError`를 발생시키는 것을 콘솔 로그에서 확인할 수 있습니다.

## 실행 방법
```bash
python main.py
```

## 시스템 구성
- **SensorTask (Process):** 0.5초마다 랜덤 수치를 발신하며, 간헐적으로 3KB의 대용량 데이터 전송을 시도합니다.
- **ControllerTask (Thread):** 센서 수치를 구독하여 임계치 초과 시 알람을 발신합니다.
- **TaskMonitor:** 웹 서버를 띄워 시스템 내부 상태를 시각화합니다.
